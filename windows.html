Windows user
We have not yet implemented scripts like 'graver', 'hilbert' or 'groebner' for Windows machines that call the appropriate functions with the appropriate options. Hence, instead of using these short cuts, you still need to do that on your own. Please find below a little dictonary of what you should call instead of the short cuts.

Replace '4ti2int32' by '4ti2int64' or '4ti2gmp' if you need a higher precision. Note that 'zsolve', and hence also 'hilbert' and 'graver' are currently only available using 32-bit integers.

Function Invocation on Windows machines
./circuits foo ./4ti2int32 circuits foo
./genmodel foo ./genmodel foo
./gensymm x y z u ./gensymm x y z u
./graver foo ./zsolve -G foo
./groebner foo ./4ti2int32 groebner foo
./hilbert foo ./zsolve -H foo
./markov foo ./4ti2int32 markov foo
./minimize foo ./4ti2int32 minimize foo
./normalform foo ./4ti2int32 normalform foo
./output --option foo ./output --option foo
./ppi n ./ppi n
./qsolve foo ./4ti2int32 qsolve foo
./rays foo ./4ti2int32 rays foo
./walk foo ./4ti2int32 walk foo
./zbasis foo ./4ti2int32 zbasis foo
./zsolve foo ./zsolve foo
